# Waypoint Guide

The Dashboard provides **List & Status** and **Install** capabilities for Waypoint, aligned with `kmeshctl waypoint`.

## List & Status

- **Waypoint** nav â†’ **List & Status**: View all Waypoints (Gateway resources with `gatewayClassName: istio-waypoint`) in the cluster.
- Filter by "all namespaces" or a specific namespace.
- Table columns: Namespace, Name, Status (Ready/Not Ready), Revision, Traffic Type; expand to view Conditions details.
- Single-item delete supported.

## Install

- **Waypoint** nav â†’ **Install Waypoint**: Create Waypoint via form.
- **Namespace**: Required, default `default`.
- **Waypoint Name**: Required, default `waypoint`; for workload granularity use e.g. `reviews-v2-pod-waypoint`.
- **Traffic Type**: Optional `service`, `workload`, `all`, `none`, corresponding to three granularities (Namespace/Service/Workload).
- **Label namespace**: When checked, sets `istio.io/use-waypoint` on the namespace.
- **Overwrite existing**, **Wait ready**, **Revision**, **Proxy Image**: Optional.

After creation, check **List & Status** and wait for Programmed to be ready.

## Custom YAML

- **Waypoint** nav â†’ **Custom YAML**: Create/update Waypoint (Gateway resource) via YAML editor.
- Default template is **Gateway** resource (`kind: Gateway`), not Deployment.

### Important: Gateway vs Deployment

**You apply a Gateway; the Deployment you see is auto-generated by Istio.**

| Resource | Created by | Description |
|----------|------------|-------------|
| **Gateway** | User / Dashboard | Your custom YAML creates/updates the Gateway resource |
| **Deployment** | Istio mesh controller | Auto-created from Gateway, config controlled by Istio |

When you run `kubectl get deploy waypoint2 -o yaml`, you see the **Deployment auto-generated by the Istio controller from the Gateway**. Its `ownerReferences` point to the Gateway, and `labels` include `gateway.istio.io/managed: istio.io-mesh-controller`. So the Deployment spec (containers, resources, volumes) differs from the Gateway YAML you edit in the Dashboardâ€”this is expected.

### How to Customize Deployment

- **Proxy Image**: Add `sidecar.istio.io/proxyImage: <image>` to Gateway `metadata.annotations`.
- **Advanced** (Istio 1.25+): Use `spec.infrastructure.parametersRef` to reference a ConfigMap and define overlay (e.g. replicas, resources) under the `deployment` key. See [Istio Gateway docs](https://istio.io/latest/docs/).

### View Your Gateway

```bash
kubectl get gateway waypoint2 -n default -o yaml
```

You should see the Gateway configuration you defined in the Dashboard.
